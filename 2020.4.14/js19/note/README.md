# 数据结构  
数据结构是一门研究非数值计算的程序设计问题中的操作**对象**,以及**他们之间的关系**和操作等相关问题的学科  
> 程序设计 = 数据结构 + 算法  

## 数据  
表象: 文字,图片,数字,音视频  
里像: 整型,字符串,布尔值等等  

## 数据结构  
> 是相互之间存在**一种或多种特定关系**的数据元素的集合  

一般来说,数据之间的关系分为  

+ 一对一  变量(Number,Boolean)
+ 一对多  (Array,String,Set)
+ 多对一  
+ 多对多 (Map)  

## 数据元素  
> 是组成数据的.有一定意义的基本单位,在计算机中通常作为整体处理  

## 数据项  
> 一个**数据元素** = 若干个**数据项**  

## 数据对象  
> 是性质相同的数据元素的集合,是数据的子集  

## 关系  
数据由数据对象组成,数据对象由多个数据元素组成,多个数据元素由多个数据项组成  

# 算法  

## 案例  
1到100,如何求和?  

代码的效率和代码执行的行数一般呈现一种正相关的关系

判断代码运行效率 往往观察其中最短的短板即可

demo1  

```JavaScript  
let sum = 0, n = 100   /* 执行1次 */  
for (let i = 1; i <= n; i++) {  /* 执行了n次 */  
    sum = sum + i  
}  
console.log(sum) /* 执行1次 */  
/* 时间复杂度为O(n) 数据量越大速度越慢 */  
```
demo2  
```JavaScript  
let sum = 0, n = 100;  /* 执行1次 */  
sum = (1 + n) * n / 2; /* 执行1次 */  
console.log(sum);   /* 执行1次 */  
/* 时间复杂度为O(1) 执行速度不受数据量的影响,是最完美的时间复杂度的情况 */
```
常见的**时间复杂度**(由快到慢)  
+ O(1)  
+ O(log₂n)  
+ O(n)  
+ O(nlog₂n)  
+ O(n²)  
+ O(n³)

# 二叉树
特殊的树,每个节点最多只能存在两个子节点.

## 完全二叉树
特殊的二叉树
+ 按照顺序排列所有的节点
+ 叶子节点只能存在在最下两层

### 满二叉树
特殊的完全二叉树
+ 按照顺序排列所有的节点
+ 叶子节点只能存在在最深一层
+ 层数必然是log₂n层
