内置对象简介

JavaScript 中的对象分为3种：自定义对象 、内置对象、 浏览器对象。

前面两种对象：是JS的基础内容，属于 ECMAScript； 第三个浏览器对象：属于JS独有，即 JS 内置的API。

内置对象：就是指这个语言自带的一些对象，供开发者使用，这些对象提供了一些常用或者最基本而必要的功能（属性和方法）。

内置对象最大的优点就是帮助我们快速开发。

Number 数字

概述

数字

(不论是小数还是整数还是负数还是正数)在JavaScript都被认为是数字类型

NaN

not a number: 不是一个数字 但是他是数字类型

Infinity/-Infinity

无穷大值

描述

  • 构造一个Number对象(极不推荐)
  • 在非构造器上下文中 (如：没有 new 操作符)，Number 能被用来执行类型转换。

语法

  • new Number(value: any): number
  • Number(value: any): number

属性

Number.NaN

特殊的“非数字”值。

特别的: NaN永远不可能等于NaN NaN === NaN OR NaN == NaN的结果是false

Number.POSITIVE_INFINITY

特殊的正无穷大值，在溢出时返回该值。

Number.NEGATIVE_INFINITY

特殊的正无穷大值，在溢出时返回该值。

Number.isNaN()

Number.isNaN(x: any): boolean

确定传递的值x是否是 NaN。

Number.isSafeInteger()

Number.isSafeInteger(x: any): boolean

确定传递的值x是否为安全整数(即Infinity) 。

Number.parseFloat()

parseFloat(value: any[, radix: number]): number

value: 待转换的值

radix: 进制 在2-32之间的数字

Number.parseInt()

parseInt(value: any[, radix: number]): number

value: 待转换的值

radix: 进制 在2-32之间的数字

Number.prototype.toFixed()

tofixed(digits: number): string

相应的小数保留digits位数,且做四舍五入 返回一个字符串类型结果

判断数字类型

判断是不是数字类型

typeof value:any

判断是不是NaN

isNaN()

当判断某个变量是不是数字的时候,往往我们需要将其变量转换成数字类型之后再做判断

isNaN不是一个专门判断数字类型的方法 -> 判断这个数字类型是不是一个真正的数字

如果我们确定这是一个数字类型,那么如果isNaN这个方法返回是true.那他一定是NaN,如果返回的是false,那他一定 是数字

  • isNaN(需要将判断的对象提前先转换数字类型) 判断是不是NaN

Null&Undefined

""&null&undefined的区别

  • "" 空字符 他是一个没有任何内容的字符串类型的文本

  • null 空: 他代表的意思是"空" 实际上已经赋值了,但是值为空

    • 我这个变量还是有值,只不过值是空(相当于在内存空间有一个专门的区域来描述这个东西是不存在)

  • undefined 未赋值的: 他代表的这个变量还没有开始赋值

    • 我这个变量根本就没有值
